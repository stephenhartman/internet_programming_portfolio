HTML Form with Javascript

For this assignment I am tasked with creating a form using HTML tags and Javascript functions. The requirements include textboxes, buttons, check boxes, dropdown menus, a reset button, and a submit button which are alldriven by a Javascript program to compute the user submitted information in a meaningful way.

I have little to no experience with Javascript and web forms, but if this task is any like previous assignments with new technologies and methods, it should a good experience. I enjoy learning new techniques and syntax because I truly feel invested in this field. Some may see this assignment as something they just need to pass a course, but I kind of like being given deadlines and goals when learning a new subject.

This project proved to be a bit of a challenge. I am particular with web design, so getting things just right takes a good bit of time. I also learned more about how the context of the attributes in HTML are more important in terms of Javascript and JQuery. Either way I feel good about my work and what I have learned through this process.

Learning Objectives:

  • Make an HTML web form which collects information using text boxes, radio buttons, check boxes and dropdown lists. Give the users options to reset the form or submit with validation.
  • Create a Javascript program to validate appropriate information, process the information if the data is valid and display an error otherwise, use conditionals and switch structures to implement the information, and present results on the screen
  • Create a flow chart which represents the decision structure.

MySQL Database

Somehow I knew my diligence would pay off, and it came in the form of the ease of designing this assignment. I already had the HTML, CSS, PHP, and Javascript laid out before me in my previous assignments, so all I had to create was a database and use CRUD functions to manipulate the data.

That being said, it was easy up to a point. One requirement of this assignment was to address SQL injection and prevent malicious queries in my forms. At first I implemented a normal SQL database with no data validation. The difficult part came in determining how to address SQL injection. There are two ways to access the database, either PDO or MySQLi, the former being compatible with many relational databases and the latter being compatible with MySQL databases. I chose MySQLi because of the requirements and the fact it was a bit easier to use knowing MySQL.

Armed with MySQLi, I was able to create queries using parameters in order to validate data. This involved building query strings in which you could pass in arguments, and those arguments would be bound to strings or integers using bind_param() and prepare(). After all the data was in the query, I used execute() on the query in order to query the database. Overall it was a good learning experience, I ran into a few issues in how to properly display data and pass along the proper queries, but in hindsight it was very straightforward compared to non-prepared statements.

Learning Objectives:

  • Create a database using MySQL, with one table used to store up to ten fields (columns).
  • Create forms using specific inputs and Javascript in order to create, modify, or delete records in the database.
  • Address SQL injection by applying the proper techniques in the SQL queries and PHP.
